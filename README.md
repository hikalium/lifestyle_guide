# hikalium の生活ガイド

このガイドは、hikaliumが日常生活を遂行する上で必要となる判断を最適化するための情報を集積したものである。

本ガイドの内容は現状や環境の変化に応じて適宜加筆訂正される。また、本バージョンには公開可能かつ（場合によっては）一般にも適用可能と思われる情報を記載する。

- [figures (hikalium only)](https://app.diagrams.net/#G1xdLdmKtGAHAJpz67Gh0wlg3kDICod0yc)

## 本ガイドにおける用語の定義

- 各要求事項の程度を示す用語については、[RFC 2119](https://datatracker.ietf.org/doc/html/rfc2119) およびその[日本語訳](https://www.nic.ad.jp/ja/tech/ipa/RFC2119JA.html)に基づいて記載する。
- 購入する物品はすべて、その製品の使用単位ごとの属性により、次の4つのいずれかに分類する。
  - 消耗品：`取得価額 < 1万円 && 使用可能期間 < 1年`
  - 高価な消耗品：`取得価額 >= 1万円 && 使用可能期間 < 1年`
  - 安価な備品：`取得価額 < 1万円 && 使用可能期間 >= 1年`
  - 備品：`取得価額 >= 1万円 && 使用可能期間 >= 1年`

## 生活を制約する資源

- 時間：ふやせない・圧倒的に足りない
- 体力：増やすのは時間がかかる・1日ごとに限りがある
- 空間：埋めるのは容易だが空けるのは困難かつ時間がかかる
- 資金：限度はあるが再生産可能

## 行動の指針

### なぜ手順を立てることが大事なのか
- なにか問題が発生したときに、つらいと感じることは通常の動作であり、それ自体に問題はない。
- しかし、問題への対処にかかる時間を最小化し、問題に関連する精神的リソースの消費を抑えることは、有限な自己という資源を効率よく利用する上で重要である。
- また、対処すべき問題が他者や社会的要因で発生する場合には、そもそもその問題自体が発生しないよう他者や社会が努力するべきである。そして、そのような問題の影響を受けている側が対処するために自己の労力を割かねばならないのは理不尽である。その前提は維持しつつも、自身の制御範囲にない主体が問題を修正するまでの間、問題の影響を受け続けるような脆弱なシステムではなく、そのような問題から受ける損害や、それに対処するリソースを最小限に抑えることができる、レジリエントなシステムを構築するほうが生産的といえる。
- この選択は、問題を発生させている他者や社会の行動を是認するものではなく、それとは直交する、運用のエンジニアリングであり、自己というシステムの安定性と機能向上をはかるものである。また、起こり得る問題への対処を事前に用意しておくことは、問題から目を背けたり、時間を割くことを拒むものではなく、むしろ問題を直視して事前に解析し、トータルでのオーバーヘッドを削減するための一つの選択可能なテクニックであり、それを引け目に感じる必要は一切ない。（コンパイル型言語とインタプリタ型言語がそれぞれ存在しているように。）

### 一般論
- 指針はあくまでも指針であって、常に従うべきものでもないし、従わなかったからといって自分を責める必要はない。
- 物事がうまく行かないと感じたときにはこの指針に従うことで、判断コストを最小限にしながら、陥りがちな罠を回避することができる。
- ただし、指針に従うと決めたならば、一度は手順を飛ばさずに評価して実行しなければならない。
- 指針に従うのが納得できないなら、指針を更新するべきである。暗黙的な例外を作ることを避け、明示的な判断に変換することが重要である。

### なんかうまくいかないとき
- まずコップいっぱいの水を飲む
  - 過集中で水分摂取を忘却している可能性があるため
- 目を閉じて1分間じっとする
  - 目を酷使しすぎていて疲れている可能性があるため
- 頭痛、目の痛み、腹痛、首の痛み、喉の痛み、発熱、その他の症状の有無を確認する
  - 「まあこの程度なら」と症状を無視するのは判断として定量的ではないので認められない
  - 自信をもって「元気である」と言えない限りは[体調が悪い時](https://github.com/hikalium/lifestyle_guide/blob/main/README.md#%E4%BD%93%E8%AA%BF%E3%81%8C%E6%82%AA%E3%81%84%E6%99%82)の手順に従ってこの問題を解決してから先に進む

### 頭が痛い時
多くの場合、目の奥がズキズキする感じの痛みである。目の酷使が要因と思われる。

- 水を3口以上飲む
- 飲み忘れている薬があれば飲む
- 横になる
- 部屋を暗くするか目を覆って暗くする
- 200秒目を閉じて横になり安静にする

### 何も手につかない / やる気が出ない
とりあえず以下をやってみる。以下の項目は手順の一部なので、それを実行することに引け目を感じる必要は一切ない。

- YouTubeでしょうもない動画をみる
- ベッドで30秒うつ伏せになりその後起き上がる（起き上がる、が重要。眠りたいと感じても、一度起き上がること）
- アイスを食べる
- 積ん読している本を数ページ読んでみる
- 「OKグーグル、面白いこと言って」
- ツイートする（投稿するだけして読まずに閉じる）
- ほしいものを買う（1万円以下）
- 甘いものを食べる
- テトリスを30分する
- マインクラフトを30分する（頭が痛い場合を除く）
- スマホやディスプレイを閉じて5分(300秒)数える

### なんだかつらい

順番に以下を達成する：

- 気温は16度以上26度以下に設定されている
- 湿度は40%以上60%以下に設定されている
- 直近30分以内に水をコップ1杯以上飲んでいる
- (省略可能) 使い捨てカイロを背中に貼っている
- 1時間以内に歯を磨いている
- 静寂ではない(音楽や動画が再生されている)
- TwitterやSlack, Gmail, Discord等、通知や情報が受動的に流入してくるものはすべて閉じられている
- スマホのおやすみモードがOnになっている

### 体調が悪い時（上記以外）
- 薬を飲み忘れていないか再確認する
- 症状に合わせて適宜薬を服用する
- だいたいの問題は睡眠を十分にとれば解決する。眠気を少しでも感じるならば、睡眠を試みるべきである。

### お風呂に入るのが面倒
- まず浴槽にお湯を張ってしまおう。物事を面倒だと感じる要因は、多くの場合、その物事を達成するまでの手順が多く存在すると少なくとも主観的には感じられているからである。それを一つずつ倒すことそれ自体を目標にすれば、1ステップで完了する物事を何回か倒すことで、元々の事柄の達成までのハードルを確実に下げることができる。これが、俗に「結果だけでなくその過程での努力も評価しよう」ということなのかもしれない。（いや、結果が大事だ、という人々に対しては、その過程の1段階それぞれが「結果」である、と主張すればよい。）
- 同様の理由で、次は服を脱ごう。もう一回着るよりかは、完全に脱ぐほうが楽だと気づくだろう。
  - 一回脱いでみて、それでももう一回着たほうが楽だという判断になったなら、おそらくそれは疲れているので、さっさと再度服を着て睡眠をしよう。
- あとは、とりあえず髪を洗うか否かは後回しにして、浴槽につかってしまおう。
  - 本当にだるかったら、30秒くらいお湯につかった後に、さっさとおふとんに戻ってもよい。だが、それも浸かってから考えることだ。
  - できれば、お湯の暖かさを感じなくなる程度まで浸かってから上がる。これにより、入浴後の急激な表面体温低下による体調悪化を避けることができる。
  - ただし、浸かりすぎると今度は体調が悪くなることがあるため、10分以上連続して浸からないようにする。
- だいたいここまでくれば、問題は解決しているはず。とにかく手順を倒すことを考えよう。

### 眠れない

- 睡眠準備（歯磨き・着替え）をすべて完了する
- 不快な匂いや音を発するものが存在しないか確認し、発見したら除去する
- 必要に応じて、適用可能であれば睡眠薬を服用する。
- 別に眠らなくてよいので、横になる。
- 本を読む。特に、積ん読されている本を読むのは有効である。

### 時間の使い方に迷ったとき
- 睡眠を最優先にしろ（はい）

### 物品の購入に迷ったとき
- 特別な事情がない限り日中に行う（夜間は判断を誤る可能性が高いため）
